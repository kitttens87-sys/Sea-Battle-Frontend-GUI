# 🌊 Sea Battle — Frontend

Фронтенд-часть веб-игры «Морской бой», выполненная в уникальной стилистике **Frutiger Aero / Windows 98**. Интерфейс сочетает в себе эстетику ретро-операционных систем, современные протоколы безопасности и real-time взаимодействие. 

Проект построен по модульному принципу и разделен на логические этапы: от стартового экрана и защищенной авторизации до полноценного боевого сеанса.

---

## 📂 Структура и архитектура проекта

Архитектура фронтенда спроектирована для обеспечения максимальной производительности при работе с real-time данными и включает в себя следующие ключевые модули:

* **Модуль аутентификации (2FA):** Обеспечивает повышенную безопасность аккаунтов. При авторизации интерфейс запрашивает не только стандартный пароль, но и временный одноразовый TOTP-код, генерируемый в мобильном приложении (например, Google Authenticator).
* **Игровой модуль:** Интерактивное поле размером 10×10 клеток. Отвечает за фазу расстановки флота (ручную игроком или автоматическую генерацию), визуализацию игровых событий, анимацию попаданий/промахов и фиксацию условий победы.

---

## 🛠 Технологический стек

* **Vue 3 (Composition API):** Основной фреймворк для построения реактивного интерфейса, разделения логики на лаконичные куски и управления состоянием приложения.
* **Vite:** Сверхбыстрый инструмент сборки для мгновенного Hot Module Replacement (HMR) в ходе разработки и оптимизации бандла для продакшена.
* **Tailwind CSS:** Утилитарный CSS-фреймворк, позволивший точно воссоздать пиксельные рамки, тени и стилистику классических десктопных окон, сохранив адаптивность.
* **Hls.js:** Библиотека для интеграции и стриминга потокового видео в качестве живого анимированного фона.
* **WebSockets (Native API):** Протокол для организации двустороннего асинхронного обмена данными с сервером для мгновенной синхронизации действий игроков во время боя.
