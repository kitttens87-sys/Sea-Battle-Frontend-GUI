## 💻 Фронтенд (Frontend)

Клиентская часть приложения представляет собой отзывчивый веб-интерфейс для симуляции настольной игры «Морской бой» в режиме реального времени. Основной упор при разработке сделан на высокую скорость отклика, точность отображения игровых событий и безопасность пользовательских сессий.
## Для запуска:
*`npm run dev`
### Ключевой функционал
* **Управление игровым процессом:** Интерактивное поле боя 10х10 с поддержкой механики Drag-and-Drop для удобной расстановки кораблей перед началом матча.
* **Синхронизация в реальном времени:** Мгновенное отображение ходов, попаданий и промахов благодаря постоянному WebSocket-соединению с бэкендом, что исключает необходимость ручного обновления страницы.
* **Двухфакторная аутентификация (2FA):** Повышенная безопасность аккаунтов. При авторизации интерфейс запрашивает не только стандартный пароль, но и временный одноразовый TOTP-код, генерируемый в мобильном приложении (например, Google Authenticator).
* **Контроль состояний (State Management):** Динамическая блокировка интерфейса в чужой ход и визуальная анимация уничтожения кораблей.

### Технологический стек
* **Vue 3 (Composition API):** Основной фреймворк для построения реактивного интерфейса и управления состоянием приложения.
* **Vite:** Быстрый инструмент сборки для оптимизации клиентских скриптов и ускорения загрузки приложения.
* **Tailwind CSS:** Утилитарный CSS-фреймворк для создания строгого, адаптивного и аккуратного интерфейса в темных корпоративных тонах.
* **WebSockets (Native API):** Протокол для организации двустороннего асинхронного обмена данными с сервером без задержек.

### Взаимодействие с сервером
Клиентская часть не хранит логику игры и состояние матча у себя. Фронтенд выступает в роли реактивного слоя отображения:
1. При клике на клетку отправляется легковесный JSON-пакет через WebSocket.
2. Сервер (Go) обрабатывает валидность хода, сверяясь с оперативной памятью (Redis).
3. Интерфейс мгновенно принимает ответный JSON-ивент и перерисовывает состояние поля для обоих игроков.
